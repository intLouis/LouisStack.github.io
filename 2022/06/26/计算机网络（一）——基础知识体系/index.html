计算机网络（一）——基础知识体系

发表于2022-06-26|更新于2022-07-06

引言：工作差不多一年，计算机网络的内容，忘得七七八八了，今天梳理一下最基本的一些计网基础知识。

这篇文章主要是给自己看的，所以这些废话也是写给自己看的，嘿嘿……

TCP/IP网络模型

这个东西就是我们天天说的TCP/IP协议，下面先讲讲整体的结构。

TCP/IP是一个协议族，是因为TCP/IP协议包括TCP、IP、UDP、ICMP、RIP、TELNETFTP、SMTP、ARP、TFTP等许多协议，这些协议一起称为TCP/IP协议。

这个协议是分层的，所以下面就逐层来进行讲解。

应用层

应用层，各位应该非常熟悉，应用层是进行生产的一个层面，也是直接与用户进行通信的。

应用层只需要专注于为用户提供服务，HTTP就是一个典型。用户层属于用户态。

应用层的主要职责是与客户端进行通信，提供一些应用程序，比如电子邮件、文件传输、远程登录等等。

应用层的话就不过多赘述，都是大家比较熟悉的部分。

传输层

应用层的数据包会传给传输层，传输层（Transport Layer）是为应用层提供网络支持的。

在传输层会有两个传输协议，分别是 TCP 和 UDP，这是我们常听到的两个协议。

TCP协议，是一种面向可靠连接，并给予字节流的传输层协议，有非常多的细节在其中，如流量控制、超时重传、拥塞控制等，这些都是为了保证数据包能可靠地传输给对方。

UDP协议，相对来说，设计较为简单，只需要发送数据包，对方是否接收得到则不做判断，传输效率相对较高，但是原生的UDP协议，显然存在丢包的缺陷，并且不可重传。

当然，UDP 也可以实现可靠传输，把 TCP 的特性在应用层上实现就可以，QQ的消息传输，就使用了UDP协议。

应用层的数据分段

应用层一般数据会比较大，因此当应用层交给传输层的数据包过大时，即超过了MSS（TCP最大报文长度），会对数据包进行分块。

这样的设计也是有一个妙处的，分担了风险：若数据包丢失，一整个进行传输需要消耗较大的网路带宽，若只是分块丢失，那么只需要重传这一小段即可，而不用重新发送整个数据包。

TCP协议中规定，每个数据分块成为一个TCP 段（TCP Segment）。

当设备作为接收方时，传输层则要负责把数据包传给应用，但是一台设备上可能会有很多应用在接收或者传输数据，因此需要用一个编号将应用区分开来，这个编号就是端口。

传输报文时会携带上端口号，以便于发送至对应的应用上。

网络层

传输层和网络层这个两个层，一开始我听懵逼的，传输层为什么叫传输层却不管传输😅。

网络层负责真正的传输。

网络层最常使用的是 IP 协议（Internet Protocol），IP 协议会将传输层的报文作为数据部分，再加上 IP 包头组装成 IP 报文，如果 IP 报文大小超过 MTU（以太网中一般为 1500 字节）就会再次进行分片，得到一个即将发送到网络的 IP 报文。
